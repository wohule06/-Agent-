# -Agent-
本项目是一款基于大语言模型（GLM-4）构建的智能代理程序。通过解析自然语言指令，自动调用天气查询工具获取实时气象数据，并结合天气状况智能推荐旅游景点。项目实现了经典的 Thought-Action-Observation Agent 运行流程，代码结构规范、注释完善，具备良好的扩展性。



# Hello-Agent 智能天气查询助手
一个基于大语言模型（GLM-4）的智能Agent助手，支持**实时天气查询** + **根据天气推荐旅游景点**，自动调用工具完成用户任务。

## ✨ 功能特性
- 智能对话交互，支持自然语言查询天气
- 调用 `wttr.in` 公共API获取**实时精准天气**
- 自动根据天气状况推荐合适的旅游景点
- 标准化Agent架构，支持扩展更多工具函数

## 📁 项目结构
```
hello-agent/
├── action.py          # 【核心】主程序入口，Agent调度与逻辑执行
├── tools封装.py       # 工具函数库（内置天气查询函数 get_weather）
├── LLM.py             # 大模型客户端封装（兼容OpenAI接口）
├── prompt.py          # Agent系统提示词配置
└── README.md          # 项目说明文档
```

## 🛠️ 环境配置
### 1. 创建并激活 Conda 环境
```bash
# 创建Python环境
conda create -n hello-agent python=3.10
# 激活环境
conda activate hello-agent
```

### 2. 安装依赖
```bash
# 网络请求库（天气查询必备）
pip install requests
```

## 🚀 运行步骤
1. **配置API密钥**
   打开 `action.py`，填写你的智谱AI API_KEY、BASE_URL等信息
2. **运行主程序**
   ```bash
   python action.py
   ```
3. **默认测试指令**
   > 你好，请帮我查询一下今天湖南的天气，如何根据天气推荐一个合适的旅游景点

## 🔧 核心功能说明
### 天气查询工具 (`get_weather`)
- 基于 `wttr.in` 免费天气API
- 支持**中文城市名**查询（自动URL编码）
- 返回格式：`城市当前天气：晴，气温 25 ℃`
- 完善的异常处理：网络错误、城市无效、数据解析失败

### Agent 工作流程
1. 接收用户请求 → 2. LLM思考分析 → 3. 调用天气工具 → 4. 生成景点推荐 → 5. 完成任务

## ⚠️ 注意事项
1. **城市名称规范**：仅支持**城市名**（如：长沙），不支持省份（如：湖南）
2. **网络要求**：确保网络可访问 `wttr.in` 服务
3. **API配置**：必须填写有效的大模型API密钥，否则无法运行
4. **依赖安装**：运行前必须安装 `requests` 库
